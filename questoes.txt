Sua importância é em torno de preservar projetos, auxiliar a equipe de desenvolvimento a trabalhar em conjunto utilizando o git e os commit, fazendo com que eles 

2-Qual a função do Git Bash?
R:Ele se torna um emulador para usar as linhas de comando do git

3-O que faz o comando git add e qual a diferença entre git add <arquivo> e git add .?
R:o comando git add adiciona as mudancas para o proximo commit que sera realizado. 
Enquanto o git add <arquivo> é usado em torno de arquivos novos e onde nao tem onde um repositório podendo ser todo tipo de alteracao como novos, modificados e os deletados. 
Ja o git add e usado para adicionar arquivos novos e mmodificados mas nao os deletados.

4-Por que utilizamos o ponto (.) no comando git add .?
R:para adicionar todos os arquivos dentro da pasta que sera commitada

5-Explique o comando git commit e por que é importante criar uma mensagem descritiva.
R:o git commit server para criar um commit do seu codigo e assim poder ter o mesmo salvo para ser usado em qualquer local. 
E necessario uma mensagem descritiva para catalogar as modificacoes feitas durante o commit 

6-Qual a função do comando git push?
R: O git push tem como funcao enviar o commit realizado anteriormente para o local que foi alocado o commit

7-Qual a importância de criar commits frequentes durante o desenvolvimento de um projeto?
R: Para sempre registrar mudancas no codigo e assim podendo sempre mostrar toda a alteracao feita sobre o codigo original

15-O que é o Flexbox no CSS e para que ele é utilizado?

16-Qual a diferença entre justify-content e align-items em um container Flexbox?
R:o justify-content alinha os itens no segmento X/Horizontal. Ja o align-items ele alinha o item nos segmentos Y/Vertical

17-Como o flex-grow funciona e qual o seu propósito no layout Flexbox?
R: Ele define a habilidade de um flex item de crescer, caso necessario. Seu proposito é a criacao de layouts flexiveis.

18-Explique a diferença entre flex-direction: column; e flex-direction: row;.
